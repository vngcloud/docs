# Repository

### What is a Repository?

A Repository is a centralized location used to store and manage Docker images. It acts as a storage space for different versions of images, where each version is identified by a unique tag. Depending on the access control settings, a Repository can be either public or private.

### What Can I Do with Repositories?

#### CREATE A REPOSITORY

To create a Repository, click the **"Create Repository"** button. You will then be asked to provide the required information such as **Repository Name**, **Access Level**, and **Quota Limit**. For more details, refer to **Create Repository**.

#### EDIT STORAGE QUOTA LIMIT

To adjust the storage quota limit of a Repository, first navigate to the Repository list screen and locate the **"Action"** column. Click the **three-dot icon** on the corresponding Repository row in the **"Action"** column, then select **"Edit Quota Limit"**. For more information, refer to **Edit Quota Limit**.

#### IMAGE MANAGEMENT

Access a Repository and click the **"Images"** tab to view the list of images stored in the Repository. You can then access a specific image to view detailed information (artifacts and tags) and copy the push/pull commands for the related artifacts/tags. For more details, refer to **Image Management**.

#### MANAGE REPOSITORY USERS

View the list of attached Repository Users by clicking the **"Repository User"** tab on the Repository detail page. From there, you can quickly create, attach, or detach a Repository User.

With an assigned role in a private Repository, a Repository User can push and pull container images, ensuring secure access to the images managed within the Repository.

For more details, refer to **Repository User**.

#### VIEW HISTORY

The Repository access history allows you to track all activities related to the Repository, including actions performed on the Repository itself and its associated Images. For more information, refer to **Repository History**.
